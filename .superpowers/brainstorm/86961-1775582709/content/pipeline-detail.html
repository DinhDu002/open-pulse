Pipeline Detail — Two-Pass Processing

Entity extraction → Graph build → Vault generation → LLM enrichment

Pass 1: Deterministic Pipeline (every 5 min)
Step 1: Extract Nodes
SQL queries on existing tables:

tools ← SELECT DISTINCT name FROM events WHERE event_type IN ('tool_call','skill_invoke','agent_spawn')

components ← SELECT * FROM components

instincts ← SELECT * FROM cl_instincts WHERE confidence > 0.3

sessions ← SELECT * FROM sessions (last 30d)

projects ← SELECT * FROM cl_projects

patterns ← derived from trigger analysis (A→B pairs)
Step 2: Extract Edges
From existing data:

triggers ← LEAD(name) OVER session window
(already in suggestion engine)

co_occurs ← tools in same session
COUNT(*) GROUP BY tool pair per session

belongs_to ← component.project → project

learned_from ← instinct.project_id → project

suggests ← suggestion.instinct_id → instinct

used_in ← events.session_id → session
Step 3: Upsert Graph
Transaction:

1. Upsert nodes
INSERT OR REPLACE INTO kg_nodes

2. Upsert edges with weight update
INSERT INTO kg_edges ... ON CONFLICT UPDATE weight = weight + new

3. Update last_updated timestamp

4. Mark kg_sync_cursor
track last processed event ID
→ next run starts from cursor
Vault Generator (every 15 min, after Pass 1)
Per Project
For each project in cl_projects:

1. Query graph: all nodes + edges for this project
(project nodes + global tools/patterns used in project sessions)

2. Filter: only include nodes with activity in this project

3. Target dir: {project_dir}/.claude/knowledge/
SHA-256 Skip
For each vault file to generate:

1. Build content from template + graph data

2. Hash content (SHA-256)

3. Compare with existing file hash
(stored in kg_vault_hashes table)

4. Skip write if hash matches

→ Typical run: 5-10% of files actually rewritten
CLAUDE.md Injection
One-time setup per project:

1. Check if @.claude/knowledge/index.md exists in CLAUDE.md

2. If not, append:
@.claude/knowledge/index.md

3. Never modify existing CLAUDE.md content

4. Add .claude/knowledge/ to .gitignore

⚠ Asks user confirmation first time
Pass 2: LLM Enrichment (on-demand or daily)
When It Runs
• Manual trigger from Open Pulse UI
• Daily timer (configurable, default off)
• After significant changes (50+ new events since last run)

User controls: enable/disable, frequency, API key config
What It Adds
Haiku analyzes graph + recent events:

• Pattern summaries
"Read→Edit→Write is a file modification workflow"

• Session insights
"Recent sessions focus on API refactoring"

• Recommendations
"Consider creating an agent for frequent debug→test loop"

• Component descriptions
Fill missing descriptions from usage context
How It Merges
Enrichments stored in kg_nodes.properties:

{ "summary": "...", "enriched_at": "..." }

Vault generator checks:
• If enriched_at exists → use LLM summary
• Otherwise → use template text

Enrichment = overlay, never replaces deterministic data
